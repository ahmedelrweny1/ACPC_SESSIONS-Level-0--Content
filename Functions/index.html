What is a Function?

A function is a reusable block of code that performs a specific task. It's like a mini-program within your program!

Why Use Functions?

  • üéØ Organize: Break complex problems into smaller, manageable pieces
  • ‚ôªÔ∏è Reuse: Write code once, use it many times
  • üßπ Simplify: Make your code cleaner and easier to understand
  • üêõ Debug: Easier to find and fix bugs in isolated functions
  • ü§ù Collaborate: Team members can work on different functions

Real-World Analogy

Think of a function like a vending machine:

  • Input: You insert money and press a button
  • Process: Machine does its work internally
  • Output: You get your snack!
C++ 
#include <iostream>
using namespace std;

// Simple function example
void greet() {
    cout << "Hello from a function!" << endl;
}

int main() {
    // Calling the function
    greet();
    greet();  // Can call multiple times!
    
    return 0;
}

/* Output:
Hello from a function!
Hello from a function!
*/

Function Structure

Anatomy of a Function

return_type function_name(parameters) {
    // Function body
    // statements
    return value;  // if not void
}

Key Components

1

Return Type

Specifies what type of value the function returns.

int getValue()       // Returns an integer
double getPrice()    // Returns a double
void display()       // Returns nothing (void)
bool isValid()       // Returns true/false
2

Function Name

Identifies the function. Use descriptive names!

// Good names
calculateAverage()
findMaximum()
printResult()

// Bad names
func1()
doStuff()
xyz()
3

Parameters (Input)

Values passed to the function for processing.

void greet(string name)              // One parameter
int add(int a, int b)                // Two parameters
void display()                        // No parameters
double calculate(int x, double y)    // Mixed types
4

Function Body

The code that gets executed when function is called.

int multiply(int a, int b) {
    int result = a * b;      // Processing
    return result;           // Return statement
}

üéØ Complete Example

#include <iostream>
using namespace std;

// Function to calculate area of rectangle
int calculateArea(int length, int width) {
    int area = length * width;
    return area;
}

int main() {
    int l = 5, w = 10;
    int result = calculateArea(l, w);
    
    cout << "Area: " << result << endl;  // Output: Area: 50
    
    return 0;
}

Void Functions

Don't return a value - just perform an action

void printWelcome() {
    cout << "Welcome!" << endl;
    // No return statement needed
}

Value-Returning Functions

Return a value that can be used

int getSquare(int n) {
    return n * n;  // Must return
}

// Usage: int x = getSquare(5);

Execution Model: How Functions Work

Understanding how a function call works is crucial for mastering programming!

üé¨ Function Call Flow

Step-by-Step Execution

#include <iostream>
using namespace std;

int double_value(int num) {
    cout << "  ‚Üí Inside function, num = " << num << endl;
    int result = num * 2;
    cout << "  ‚Üí Calculated result = " << result << endl;
    return result;
}

int main() {
    cout << "Step 1: Start main()" << endl;
    
    int x = 5;
    cout << "Step 2: x = " << x << endl;
    
    cout << "Step 3: Calling double_value(x)" << endl;
    int y = double_value(x);  // Control jumps to function
    
    cout << "Step 4: Back in main, y = " << y << endl;
    cout << "Step 5: End main()" << endl;
    
    return 0;
}

/* Output:
Step 1: Start main()
Step 2: x = 5
Step 3: Calling double_value(x)
  ‚Üí Inside function, num = 5
  ‚Üí Calculated result = 10
Step 4: Back in main, y = 10
Step 5: End main()
*/

Key Points

  • Call: Program jumps from caller to function
  • Execute: Function body runs with provided arguments
  • Return: Control returns to caller with result (if any)
  • Continue: Program continues from where it left off

Pass by Value vs Pass by Reference

üìã Pass by Value (Copy)

A copy of the variable is passed. Original unchanged.

#include <iostream>
using namespace std;

void modifyValue(int x) {
    x = 100;  // Only modifies the copy
    cout << "Inside: " << x << endl;
}

int main() {
    int num = 5;
    cout << "Before: " << num << endl;
    
    modifyValue(num);
    
    cout << "After: " << num << endl;
    
    return 0;
}

/* Output:
Before: 5
Inside: 100
After: 5    ‚Üê Original unchanged!
*/

üîó Pass by Reference

The actual variable is passed. Changes affect original.

#include <iostream>
using namespace std;

void modifyReference(int &x) {  // Note the &
    x = 100;  // Modifies the original!
    cout << "Inside: " << x << endl;
}

int main() {
    int num = 5;
    cout << "Before: " << num << endl;
    
    modifyReference(num);
    
    cout << "After: " << num << endl;
    
    return 0;
}

/* Output:
Before: 5
Inside: 100
After: 100   ‚Üê Original changed!
*/

üéÆ Interactive Demo: Pass by Value vs Reference

üìã

Use Pass by Value When:

  • You don't want to modify the original
  • Working with small data (int, char, bool)
  • Need to keep original safe
üîó

Use Pass by Reference When:

  • You want to modify the original
  • Working with large data (avoid copying)
  • Returning multiple values

üí° Practical Example: Swap Function

#include <iostream>
using namespace std;

// Pass by reference to swap values
void swap(int &a, int &b) {
    int temp = a;
    a = b;
    b = temp;
}

int main() {
    int x = 10, y = 20;
    
    cout << "Before swap: x = " << x << ", y = " << y << endl;
    
    swap(x, y);  // Pass by reference
    
    cout << "After swap: x = " << x << ", y = " << y << endl;
    
    return 0;
}

/* Output:
Before swap: x = 10, y = 20
After swap: x = 20, y = 10
*/

Variable Scope

Scope determines where a variable can be accessed in your program.

üè† Local Scope

Variables declared inside a function

void myFunction() {
    int localVar = 10;  // Local to myFunction
    cout << localVar;
}

int main() {
    // cout << localVar;  ‚ùå ERROR!
    // localVar doesn't exist here
    
    myFunction();  // localVar exists here
    return 0;
}

üåç Global Scope

Variables declared outside all functions

int globalVar = 100;  // Global variable

void myFunction() {
    cout << globalVar;  // ‚úÖ Can access
}

int main() {
    cout << globalVar;  // ‚úÖ Can access
    myFunction();
    return 0;
}

üìù Complete Scope Example

#include <iostream>
using namespace std;

int globalCounter = 0;  // Global - accessible everywhere

void incrementCounter() {
    int localStep = 1;  // Local to this function
    globalCounter += localStep;
    cout << "Counter inside function: " << globalCounter << endl;
}

int main() {
    cout << "Initial counter: " << globalCounter << endl;
    
    incrementCounter();
    incrementCounter();
    
    cout << "Final counter: " << globalCounter << endl;
    
    // cout << localStep;  ‚ùå ERROR! localStep is not accessible here
    
    return 0;
}

/* Output:
Initial counter: 0
Counter inside function: 1
Counter inside function: 2
Final counter: 2
*/
‚ö†Ô∏è Global Variables - Use with Caution!
  • Risk: Can be modified from anywhere (hard to track bugs)
  • Risk: Name conflicts with local variables
  • Risk: Makes code harder to understand and maintain
  • Best Practice: Prefer passing parameters instead of globals
‚úÖ

Local Variables Are Better

  • Contained within function
  • No side effects
  • Easier to debug
  • Memory auto-managed
‚ö†Ô∏è

Global Variables Should Be

  • Used sparingly
  • For truly global data
  • Constants (const keyword)
  • Well documented

Advanced Function Features

1

Function Overloading

Multiple functions with the same name but different parameters.

#include <iostream>
using namespace std;

// Same function name, different parameters
int add(int a, int b) {
    return a + b;
}

double add(double a, double b) {
    return a + b;
}

int add(int a, int b, int c) {
    return a + b + c;
}

int main() {
    cout << add(5, 10) << endl;        // Calls int version ‚Üí 15
    cout << add(5.5, 10.3) << endl;    // Calls double version ‚Üí 15.8
    cout << add(1, 2, 3) << endl;      // Calls 3-param version ‚Üí 6
    
    return 0;
}

Overloading Rules:

  • Different number of parameters
  • Different types of parameters
  • Return type alone is NOT enough
2

Default Parameters

Provide default values for parameters if not specified.

#include <iostream>
using namespace std;

// Default parameters (must be rightmost)
void greet(string name, string message = "Hello") {
    cout << message << ", " << name << "!" << endl;
}

void printNumber(int n, int times = 1) {
    for(int i = 0; i < times; i++) {
        cout << n << " ";
    }
    cout << endl;
}

int main() {
    greet("Alice");              // Uses default: "Hello, Alice!"
    greet("Bob", "Hi");          // Custom: "Hi, Bob!"
    
    printNumber(5);              // Prints: 5
    printNumber(5, 3);           // Prints: 5 5 5
    
    return 0;
}
‚ö†Ô∏è Default Parameter Rules:
  • Must be rightmost parameters
  • Can't skip middle parameters when calling
3

Function Prototypes (Forward Declaration)

Declare function before definition to use it anywhere.

#include <iostream>
using namespace std;

// Function prototypes (declarations)
int add(int a, int b);
void printResult(int result);

int main() {
    int sum = add(5, 10);     // Can use before definition
    printResult(sum);
    
    return 0;
}

// Function definitions (can be after main)
int add(int a, int b) {
    return a + b;
}

void printResult(int result) {
    cout << "Result: " << result << endl;
}

Why Use Prototypes?

  • Call functions before they're defined
  • Organize code better (main first)
  • Required for mutual recursion
  • Separate declaration from implementation

2. Array Operations

Write functions to find sum and average of an array using pass by reference.

Show Solution 
void arrayStats(int arr[], int size, 
                int &sum, double &avg) {
    sum = 0;
    for(int i = 0; i < size; i++) {
        sum += arr[i];
    }
    avg = (double)sum / size;
}

// Usage:
int main() {
    int numbers[] = {10, 20, 30, 40, 50};
    int total;
    double average;
    
    arrayStats(numbers, 5, total, average);
    
    cout << "Sum: " << total << endl;      // 150
    cout << "Average: " << average << endl; // 30
    
    return 0;
}
